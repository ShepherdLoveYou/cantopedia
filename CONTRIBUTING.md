# Contributing to Cantopedia

> Community contribution flow opens in **v0.2**. This file is a placeholder
> for v0.1-alpha.

## What you can do today

- **File an issue** if you see:
  - A factual error in a recipe, ingredient, or sauce
  - A bad translation between 粵 / 中 / English
  - A potential copyright / license problem with an image or text
  - A missing source citation
- **Star or fork** the repo if you find it useful.

## What we are *not* yet ready for

- Recipe submissions
- Photo submissions
- Translation contributions
- Code PRs (beyond trivial typo fixes)

We are still defining the contribution standards — schema constraints,
license screening, citation requirements, and review SLA. Opening the
floodgates before those are in place tends to create more cleanup than value.

## When v0.2 opens, you can expect

- A clear **dish submission template** in `.github/ISSUE_TEMPLATE/`
- A **license screening checklist** for any contributed image
- A **citation discipline** doc: every claim must trace to a source
- A **review SLA** so contributors know what response time to expect
- **CODE_OF_CONDUCT.md**

## Reaching out

For licensing questions: open an issue tagged `license`.
For schema or technical questions: open an issue tagged `data` or `pipeline`.
For everything else: open an issue tagged `discussion`.

Thank you for your patience.
